# FamilyMind 2.0 — Implementation Plan

> For Claude: REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

Goal: Build a structured parenting platform with flexible content packaging, multi-model monetization, and a layered architecture that ships in phases.

Architecture: Next.js 16 App Router with server components and server actions. PostgreSQL via Supabase (managed DB + auth + storage). Prisma ORM. Stripe for payments. Deployed on Vercel.

Tech Stack: Next.js 16, React 19, TypeScript 5, Prisma, Supabase (Postgres + Auth + Storage), Stripe, Tailwind CSS 4, shadcn/ui, Resend (email), Vercel.

---

## Critical Technical Decisions (Stated, Not Asked)

These are opinionated choices. Override any you disagree with.

| Decision          | Choice                        | Rationale                                                                                      |
|-------------------|-------------------------------|------------------------------------------------------------------------------------------------|
| Database          | Supabase (managed Postgres)   | Auth, DB, storage, and realtime in one service. Free tier for dev. Row-level security for B2B. |
| ORM               | Prisma                        | Most mature Next.js ORM. Type-safe. Migrations. Introspection.                                 |
| Auth              | Supabase Auth                 | Already included. Social login, magic link, email/password. Handles sessions.                  |
| Payments          | Stripe                        | Industry standard. Handles subscriptions, one-time, coupons, invoicing. Full EU/DK support.    |
| Video hosting     | External URLs (agnostic)      | Store video URLs, not files. Works with NewZenler embeds now, Mux/Bunny later. No migration.   |
| UI components     | shadcn/ui                     | Not a dependency -- copied into project. Full control. Accessible. Tailwind-native.            |
| Email             | Resend + React Email          | Modern, developer-friendly. Transactional email with React components.                         |
| Deployment        | Vercel                        | Native Next.js host. Zero config. Preview deployments. Edge functions.                         |
| Admin             | Built-in, role-based          | No external CMS. Admin pages behind role check. Same codebase. Simpler.                        |

## Phase 1a: Content Layer + Product/Monetization Layer

### Task 1.1: Database Schema — Core Models

What: Define the complete Prisma schema for content, products, users, and entitlements.

Models:

User
  - id (uuid, from Supabase Auth)
  - email
  - name
  - role (USER, ADMIN)
  - organizationId (nullable -- B2B ready)
  - onboardingCompleted (boolean)
  - createdAt, updatedAt

Organization (B2B-ready, unused in Phase 1)
  - id
  - name
  - slug
  - createdAt

ContentUnit
  - id
  - title
  - description
  - slug (unique)
  - mediaType (VIDEO, AUDIO, PDF, TEXT)
  - mediaUrl (external URL -- provider agnostic)
  - thumbnailUrl
  - durationMinutes
  - difficulty (INTRODUCTORY, INTERMEDIATE, ADVANCED)
  - isStandalone (boolean)
  - isFree (boolean)
  - publishedAt (nullable -- null means draft)
  - createdAt, updatedAt

ContentTag
  - id
  - name (unique, e.g. "sleep", "boundaries")
  - slug

ContentUnitTag (join table)
  - contentUnitId
  - tagId

AgeRange (on ContentUnit)
  - ageMin (int, months)
  - ageMax (int, months)

Product
  - id
  - title
  - description
  - slug
  - type (SUBSCRIPTION, COURSE, SINGLE, BUNDLE)
  - priceAmountCents (int)
  - priceCurrency (default "DKK")
  - stripeProductId
  - stripePriceId
  - isActive (boolean)
  - createdAt, updatedAt

CourseLesson (join: Product <-> ContentUnit for COURSE products)
  - id
  - productId (the course)
  - contentUnitId
  - position (int, ordering)

BundleItem (join: Product <-> Product for BUNDLE products)
  - id
  - bundleProductId
  - includedProductId

Entitlement
  - id
  - userId
  - organizationId (nullable -- B2B ready)
  - productId
  - source (PURCHASE, SUBSCRIPTION, GIFT, B2B_LICENSE)
  - status (ACTIVE, EXPIRED, CANCELLED)
  - stripeSubscriptionId (nullable)
  - expiresAt (nullable -- null means permanent)
  - createdAt

DiscountCode
  - id
  - code (unique)
  - type (PERCENTAGE, FIXED_AMOUNT)
  - value (int -- percentage or cents)
  - maxUses (nullable)
  - currentUses
  - validFrom
  - validUntil (nullable)
  - applicableProductId (nullable -- null means all products)
  - isActive (boolean)

Steps:
1. Write full Prisma schema in prisma/schema.prisma
2. Run: npx prisma migrate dev --name init (creates all tables)
3. Create seed file prisma/seed.ts with test data (1 admin user, 5 content units, 2 courses, 1 bundle)
4. Run seed: npx prisma db seed
5. Verify in Supabase dashboard: all tables populated
6. Commit

## Non-Functional Requirements

### Security
- All admin routes behind role check
- Stripe webhook signature verification
- Input validation with Zod on all server actions
- CSRF protection via Next.js server actions (built-in)
- GDPR: user data export + deletion capability (add in Phase 1a)

### Performance
- Server components by default (no unnecessary client JS)
- Database queries with Prisma includes, avoid N+1
- Image/thumbnail optimization via Next.js Image
- Video served from external provider (no self-hosting bandwidth)

### Monitoring
- Vercel Analytics (built-in)
- Error tracking: Sentry
- Custom event tracking for retention KPIs (simple: log to DB table, build dashboard later)
