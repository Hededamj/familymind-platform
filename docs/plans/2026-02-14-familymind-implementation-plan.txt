# FamilyMind 2.0 — Implementation Plan (v2)

> For Claude: REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

Goal: Build a structured parenting platform with flexible content packaging, multi-model monetization, and a layered architecture that ships in phases.

Architecture: Next.js 16 App Router with server components and server actions. Service layer pattern: all business logic in lib/services/ (framework-agnostic), server actions are thin wrappers. This enables a future native app (Capacitor) to call the same logic via API routes.

Tech Stack: Next.js 16, React 19, TypeScript 5, Prisma, Supabase (Postgres + Auth + Storage), Stripe, Bunny.net Stream (video), Tailwind CSS 4, shadcn/ui, Resend (email), Vercel.

---

## Critical Technical Decisions

| Decision          | Choice                        | Rationale                                                                                      |
|-------------------|-------------------------------|------------------------------------------------------------------------------------------------|
| Database          | Supabase (managed Postgres)   | Auth, DB, storage, and realtime in one service. Row-level security for B2B.                    |
| ORM               | Prisma                        | Most mature Next.js ORM. Type-safe. Migrations. Introspection.                                 |
| Auth              | Supabase Auth                 | Email/password + Google OAuth + Apple Sign-In. Handles sessions.                               |
| Payments          | Stripe                        | Subscriptions, one-time, coupons, invoicing. Full EU/DK support.                               |
| Video hosting     | Bunny.net Stream              | EU-based (GDPR), affordable, HLS adaptive streaming, token-based protection. Can migrate to Mux later. |
| UI components     | shadcn/ui                     | Copied into project. Full control. Accessible. Tailwind-native.                                |
| Email             | Resend + React Email          | Transactional email with React components.                                                     |
| Deployment        | Vercel                        | Native Next.js host. Preview deployments. Cron jobs.                                           |
| Admin             | Built-in, role-based          | Admin pages behind role check. Same codebase. No external CMS.                                 |
| Architecture      | Service layer pattern         | lib/services/ = pure business logic. Server actions and future API routes are thin wrappers.    |
| Future app        | Capacitor (later)             | Wraps consumer-facing web pages in native shell. No in-app purchases (web checkout only).      |

---

## Architectural Pattern: Service Layer

All business logic lives in lib/services/. No Next.js imports. No request/response objects. Pure functions that take data and return data.

```
lib/services/
  content.service.ts      -- CRUD, publishing, filtering
  product.service.ts      -- CRUD, Stripe sync, pricing
  entitlement.service.ts  -- access checks, grant/revoke
  journey.service.ts      -- CRUD, progression, check-ins
  onboarding.service.ts   -- quiz logic, recommendations
  engagement.service.ts   -- notifications, ladder, milestones
  checkout.service.ts     -- Stripe sessions, discount validation
  progress.service.ts     -- content/course/journey progress
  user.service.ts         -- profile, preferences, sync
```

Server actions (app/admin/*/actions.ts) call services. Future API routes (app/api/v1/*) call the same services. Zero duplicated logic.

## Non-Functional Requirements

### Security
- All admin routes behind role check
- Stripe webhook signature verification
- Input validation with Zod on all server actions
- CSRF protection via Next.js server actions (built-in)
- Bunny.net signed URLs for video content protection
- GDPR: user data export + deletion capability (add in Phase 1a)

### Performance
- Server components by default (no unnecessary client JS)
- Database queries with Prisma includes, avoid N+1
- Image/thumbnail optimization via Next.js Image
- Video served from Bunny.net CDN (EU edge, low latency for DK users)

### Mobile-First
- All user-facing pages designed mobile-first
- Dashboard, browse, content, journey, checkout -- all optimized for mobile
- Admin pages: desktop-first (admin is not used on mobile)
- Prepares for Capacitor wrapper (future native app)

### Monitoring
- Vercel Analytics (built-in)
- Error tracking: Sentry
- Custom event tracking for retention KPIs (log to DB table, build dashboard later)
