   - Empty reasonSlugs → null
   - Look up CancellationReason ids by slugs; empty → null

   - Walk candidates; first eligible one returned as EligibleOffer DTO

```typescript
await stripe.subscriptions.update(

)
```

7. `applyPauseOffer(userId, entitlementId, months, surveyId, offerId)` — internal. Call `pauseSubscription({ userId, entitlementId, months })` and return its `resumesAt` as the Date. Do NOT create the acceptance row here — acceptOffer() does that centrally.

   - Load offer with findUniqueOrThrow
   - Resolve stripeAccountId via helper
   - Branch by offerType:

     - PAUSE: cast `(offer.pauseMonths ?? 1) as 1|2|3`, call applyPauseOffer → expiresAt = result
     - SUPPORT / CONTENT_HELP / NONE: no Stripe call for the offer itself
   - Auto-reverse cancel_at_period_end for all offer types EXCEPT PAUSE:
     - Retrieve sub, cast as `{ cancel_at_period_end: boolean }`, if true call reverseCancelAtPeriodEnd and set cancelReversed = true
   - Create RetentionOfferAcceptance row (offerId, surveyId, userId, offerType: offer.offerType, expiresAt)
   - Return `{ accepted: true, acceptanceId, expiresAt, cancelReversed }`
