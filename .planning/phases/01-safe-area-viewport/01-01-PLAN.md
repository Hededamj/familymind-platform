**app/layout.tsx — Add viewport metadata:**

In the `generateMetadata()` function, add a `viewport` export. In Next.js 14+, viewport is exported separately from metadata. Add this export ABOVE or BELOW generateMetadata:

```typescript
export const viewport: Viewport = {
  width: 'device-width',
  initialScale: 1,
  viewportFit: 'cover',
}
```

Add the import at the top of the file:
```typescript
import type { Metadata, Viewport } from "next";
```

This is the standard Next.js way to set viewport-fit=cover. It renders as `<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">`.

**components/layout/app-topbar.tsx — Add safe-area-inset-top:**

On the `<header>` element:
1. Remove `h-14` from className (height will be controlled by style prop).
2. Add a style prop with paddingTop and height:
   ```
   style={{
     paddingTop: 'env(safe-area-inset-top, 0px)',
     height: 'calc(3.5rem + env(safe-area-inset-top, 0px))',
   }}
   ```
3. Keep ALL other classes unchanged: `sticky top-0 z-40 flex items-center justify-between border-b border-[var(--border)] bg-[var(--background)] px-4 md:hidden`

The visible content (brand name, back button, bell icon) stays centered in the lower 56px via `items-center` on the flex container — the paddingTop pushes content down from the notch area.
