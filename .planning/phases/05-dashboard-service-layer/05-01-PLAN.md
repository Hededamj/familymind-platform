<task type="auto" tdd="true">
  <name>Task 1: Add getCheckInPrompt function</name>
  <files>lib/services/dashboard.service.ts</files>
  <behavior>
    - Active journey user: returns "Hvordan gar det med [journey.title]?" (uses journey title)
    - Active journey user with current phase: returns "Hvordan gar det med [phase.title]?" (uses phase title when available, journey title as fallback)
    - User with primaryChallengeTagId but no active journey: returns "Hvordan gar det med [ContentTag.name]?" (resolves tag name from DB)
    - New user (no profile, no journey): returns "Velkommen! Hvordan har du det i dag?"
    - Completed journey user: returns "Tillykke! Hvad er dit naeste mal?"
    - User with no journey and no challenge tag: returns "Hvordan har du det i dag?"
  </behavior>
  <action>
    Add `getCheckInPrompt(userId: string): Promise<string>` to dashboard.service.ts.

    Implementation (per CHECKIN-01 and CHECKIN-03):
    1. Call `getUserActiveJourney(userId)` (already imported).
    2. If active journey exists and has currentDay with phase:
       - Return `Hvordan gar det med ${activeJourney.currentDay.phase.title}?`
       - Fallback to `Hvordan gar det med ${activeJourney.journey.title}?` if phase title missing.
    3. If no active journey, check for completed journey:
       - `prisma.userJourney.findFirst({ where: { userId, status: 'COMPLETED' } })` (already done in getDashboardState, but this function is standalone).
       - If found: return `Tillykke med at gennemfore dit forlob! Hvad er dit naeste mal?`
    4. If neither, load `prisma.userProfile.findUnique({ where: { userId }, select: { primaryChallengeTagId: true } })`.
       - If primaryChallengeTagId exists: resolve tag name via `prisma.contentTag.findUnique({ where: { id: primaryChallengeTagId } })`, return `Hvordan gar det med ${tag.name}?`
    5. Check if user has a profile at all (onboarding completed):
       - If no profile: return `Velkommen! Hvordan har du det i dag?`
       - If profile but no tag: return `Hvordan har du det i dag?`

    Use Danish characters properly in the actual strings (use the actual characters: aa->a with ring, ae->ae ligature, oe->o with stroke). The behavior section above uses ASCII for compatibility.

    Export the function. Do NOT modify existing getDashboardState yet (that happens in Plan 02).
  </action>
  <verify>
    <automated>npx tsx -e "import { getCheckInPrompt } from './lib/services/dashboard.service'; console.log(typeof getCheckInPrompt === 'function' ? 'PASS: exported' : 'FAIL')"</automated>
  </verify>
  <done>getCheckInPrompt is exported, handles all 5 state branches (active journey, completed journey, has challenge tag, new user, no-tag user), returns Danish prompt strings</done>
</task>
