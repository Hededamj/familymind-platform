**Step 1: Extend getCourseProgress in lib/services/progress.service.ts**

Modify the `getCourseProgress` function to accept an additional parameter `productModuleCount` (optional, to avoid an extra query) and return two additional fields. Since the function already loads all lessons with contentUnit, we can compute totalDurationMinutes from the loaded data. For chapterCount, the caller (course page) already has `product.modules.length`, so we pass it through.

Actually, simpler approach: add `chapterCount` and `totalDurationMinutes` computation directly into `getCourseProgress`. This requires one extra query for modules count. Update the function:

```typescript
export async function getCourseProgress(userId: string, productId: string) {
  const [lessons, moduleCount] = await Promise.all([
    prisma.courseLesson.findMany({
      where: { productId },
      include: {
        contentUnit: {
          include: {
            userProgress: { where: { userId } },
          },
        },
      },
      orderBy: { position: 'asc' },
    }),
    prisma.courseModule.count({ where: { productId } }),
  ])

  const totalLessons = lessons.length
  const completedLessons = lessons.filter(
    (l) => l.contentUnit.userProgress[0]?.completedAt
  ).length
  const percentComplete =
    totalLessons > 0
      ? Math.round((completedLessons / totalLessons) * 100)
      : 0

  const chapterCount = moduleCount
  const totalDurationMinutes = lessons.reduce(
    (sum, l) => sum + (l.contentUnit.durationMinutes ?? 0),
    0
  )

  return {
    totalLessons,
    completedLessons,
    percentComplete,
    chapterCount,
    totalDurationMinutes,
    lessons: lessons.map((l) => ({
      id: l.id,
      contentUnitId: l.contentUnit.id,
      title: l.contentUnit.title,
      slug: l.contentUnit.slug,
      position: l.position,
      started: !!l.contentUnit.userProgress[0],
      completed: !!l.contentUnit.userProgress[0]?.completedAt,
    })),
  }
}
```

Key changes:
- Add `prisma.courseModule.count({ where: { productId } })` in a `Promise.all` with the existing lessons query (no extra round trip)
- Add `chapterCount` (from module count) to return object
- Add `totalDurationMinutes` (sum of `durationMinutes ?? 0` from already-loaded lessons) to return object
- The `durationMinutes` field is nullable `Int?` — always coerce with `?? 0` to avoid NaN

Also update `getUserInProgressCourses` to include `chapterCount` and `totalDurationMinutes` in its return shape for consistency. In the for-loop (around line 103), after computing `percentComplete`, add:
